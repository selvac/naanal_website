Challenges in IT infrastructure

Today, enterprise is struggling in one way or the other in order to meet business needs. Key problems includes:

Every enterprise and every business group within an enterprise has its own unique IT requirement.

IT end users want to simply use applications as opposed to installing and managing them.

IT administrators want to offer a well-tested set of on-demand self-service applications to dramatically reduce their support burden

The problem can be better understood with the toy making analogy. If an end user asks for a shape say pyramid, then Admin creates a pyramid shape out of plastic. If a user no longer needs pyramid shape and need cube, then the previously built pyramid is waste and fresh raw plastic has to be procured to build this new shape.

Getting same shape consistently is another challenge. So this will cost the company significantly in terms of time and money.

Naanal's Solution

The solution is to convert the raw plastic into building “Blocks” of different shapes and colours. Then Admin uses these bricks to build a pyramid for the user. If the user wants cube, then the same pyramid can be re-purposed to cube . Also the cube can be replicated so the same shape can be obtained consistently by anyone. This will save significant cost and time. Abstraction, Orchestration & Automation is the key here.

Naanal does exactly the same thing - Abstract every IT resource such as compute, storage, networking, database etc., as building “Blocks”. Then Admin creates “Stack” a.k.a template that describes infrastructure requirement (compute, storage,networking), software dependencies etc., to deploy the application stack or platform stack
